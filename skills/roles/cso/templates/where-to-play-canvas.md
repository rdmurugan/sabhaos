# Where-to-play / how-to-win canvas — template

The single most important strategic artifact at operator-stage. One page; updated annually; referenced before every major resource-allocation decision.

---

## Section 1 — Where to play (1 page max)

```
PRIMARY MARKET (where we win first):

Segment:
- Customer type: [   ] (e.g., B2B SaaS companies, 50-500 employees)
- Industry / vertical: [   ]
- Geography: [   ]
- Stage: [   ] (e.g., post-PMF, growing)
- Specific persona to sell to: [   ] (e.g., VP Engineering)
- Specific buying trigger: [   ] (e.g., "just raised Series A")

Use case (the JTBD):
- The job: [what they're hiring our product to do]
- The trigger: [what makes them hire it]
- The alternative: [what they're hiring today]

Distribution:
- How they discover us: [   ]
- How they buy: [self-serve / sales-led / channel]
- Average sales cycle: [   ]
- Average ACV: [   ]
```

## Section 2 — Adjacent markets (next 12-24 months)

```
ADJACENT 1 (after we lock primary):
- Segment: [   ]
- Why this is the right next move: [   ]
- Entry trigger: [primary needs to hit X first]

ADJACENT 2 (after Adjacent 1):
- Segment: [   ]
- Why: [   ]
- Entry trigger: [   ]
```

## Section 3 — Markets we WON'T serve

```
NOT-PLAYING (the explicit no list):

We will NOT pursue:
- [Specific segment] because [reason]
- [Specific use case] because [reason]
- [Specific geography] because [reason]
- [Specific buying motion] because [reason]

This list matters because every quarter someone proposes pursuing one of these.
Saying NO explicitly + in writing prevents the dilution.
```

---

## Section 4 — How to win (in the primary market)

```
THE WEDGE — the specific thing only we do for this segment:

[Specific + true + defensible for 2-3 years]

Example: "We're the only [thing] that integrates with [specific tool the 
segment uses universally] AND provides [specific capability no competitor 
has yet]."
```

```
THE UNFAIR ADVANTAGE — why we'll win over competitors:

Product:        [what we have they don't]
Distribution:   [what access we have they don't]
Operations:     [what we can do operationally that competitors can't]
Brand / team:   [what credibility / network we have]
Data / network: [what we have access to that compounds]

If you can't fill in at least 2 of these credibly, you don't have an unfair 
advantage yet. The work is building one.
```

```
THE KEY METRICS — what tells us we're winning:

3-month metric: [leading indicator — e.g., demo-to-close rate]
6-month metric: [intermediate — e.g., paying customer count in segment]
12-month metric: [outcome — e.g., $X ARR from segment]

If we hit these, we're winning the way we planned.
If we don't, the strategy needs adjustment, not just execution.
```

---

## Section 5 — The big bets (this year)

Pick 1-3 big bets that, if they pay off, transform the trajectory. Each gets:

```
BET 1: [name]

The bet statement: "We are betting that [condition], and if true, [outcome]. 
We're putting [resources] behind this for [duration]."

What would have to be true for this to fail: [3-5 failure modes]
Expected value: [upside × probability − downside × probability]
Check-in cadence: [monthly review of the leading indicator]
Kill threshold: [if leading indicator shows X by Y date, we walk]
```

Repeat for BET 2, BET 3.

**Operator discipline:** 1-3 big bets per year. More than that is unfocus.

## Section 6 — Resource allocation

```
PRIMARY MARKET (% of resources):
- Engineering: [   ]%
- Sales / GTM: [   ]%
- Marketing: [   ]%
- Customer success: [   ]%

BIG BETS (% of resources):
- Bet 1: [   ]%
- Bet 2: [   ]%
- Bet 3: [   ]%

INFRASTRUCTURE / OPERATIONS:
- [   ]%

EXPLORATORY / LEARNING:
- [   ]% (small but non-zero)

TOTAL: 100%
```

If big bets are >40% of resources, you're betting the company. Decide deliberately.

## Section 7 — Scenarios

```
BASE SCENARIO (50% probability):
[Default operating plan; what we expect to happen]

UPSIDE SCENARIO (25%):
[What we do if things break well]

DOWNSIDE SCENARIO (25%):
[What we do if things break badly]

For each:
- Trigger that signals which scenario we're in (by month 3 / 6)
- Resource implications
- Key decisions if we recognize we're in it
```

Cross-reference: CFO `templates/runway-model.md` for cash impact of each scenario.

## Section 8 — The 5 strategic questions (recurring check-in)

Every quarterly review, answer:

```
1. Are we still in the right primary market?
   - Evidence: [   ]
   - Confidence: [high / medium / low]

2. Is our wedge still defensible?
   - What competitors have done in the last 90 days: [   ]
   - Have they eroded our wedge? [Y/N]

3. Are our big bets tracking?
   - Bet 1: [on/off track]
   - Bet 2: [on/off track]
   - Bet 3: [on/off track]

4. Are we executing the strategy or drifting?
   - Are we doing things on our "not-playing" list? Why?
   - Are we spending resources on non-priority initiatives?

5. What should change?
   - Tactical adjustments (continue executing same strategy)
   - Strategic adjustments (refine where-to-play / how-to-win)
   - Major adjustments (pivot or reposition)
```

## When to update the canvas

| Trigger | Update? |
|---|---|
| Quarterly check-in | Yes — quick refresh, validate or adjust |
| Annual planning | Yes — full rewrite, every section |
| Major market shift (competitor consolidation, new regulation, etc.) | Yes — focused on affected sections |
| Founder change in conviction | Yes — surface and re-decide |
| Investor / board strategic review | Yes — use the canvas as the artifact |
| New product line consideration | Yes — does it fit where-to-play? |

---

## Anti-patterns

| Pattern | Why bad |
|---|---|
| Canvas is just aspirations | Strategy needs to be lived, not stated |
| Canvas is 10 pages | One page forces clarity |
| Canvas doesn't have a "not playing" list | You're not making real trade-offs |
| Canvas has 8 big bets | That's not focused; that's a wishlist |
| Canvas not referenced between annual planning | Strategy needs operating cadence |
| Canvas + reality diverge by month 6 | Update the canvas OR adjust reality. Don't ignore the gap. |

---

## When this template is overkill

For pre-PMF startups still searching for fit: a strategy canvas is premature. Use a one-paragraph "what we're trying to build, for whom, and why" instead. Add the canvas once you've found PMF and are scaling.

For solo operators / lifestyle businesses: full canvas is overkill. The 3-question filter (where to play, how to win, what to give up) in a paragraph is enough.

For pre-Series A operator-stage with funding: this template is right.
