# Decision memo — template

For founder-level decisions worth filing. The act of writing this out clarifies the decision more than any framework.

---

## Header

```
Decision title:          [one line: what's being decided]
Date:                    [   ]
Decider:                 [   ]
Status:                  [draft / decided / in-progress / done]
Reversibility:           [two-way door / one-way door / partially reversible]
```

---

## Context (1 paragraph)

```
What's happening? What changed? Why does this decision need to be made now 
(vs. later)?

[2-3 sentences]
```

---

## The decision

```
Specifically, the decision is between:

OPTION A: [description]
OPTION B: [description]  
OPTION C: [description if 3+]

Each option has its own logic. The reason this is hard is [why].
```

## The irreducible question

```
After integrating the functional perspectives, the underlying question this 
decision turns on is:

[The single question of belief, judgment, or values that resolves the decision.]

If [belief X is true] → Option A.
If [belief Y is true] → Option B.

This is a question only the decider can answer.
```

## Why file a decision memo?

For decisions that pass the "would I want my future self to remember this clearly?" test:

```
- Pivots
- Major hires (especially executives)
- Major firings (especially executives)
- Fundraising (any round)
- Major customer commitments
- Strategic partnerships
- Pricing or business-model changes
- M&A consideration
- Founder agreements / equity changes
- Sabbatical / departure decisions
```

The act of writing it forces clarity. The record helps future-you remember the reasoning.

---

## When NOT to file

Most decisions don't need a memo. Routine operational choices are made and forgotten. Reserve this template for the 5-10 decisions per year that shape the company's trajectory.

If you find yourself writing memos weekly, something is wrong — either the work is too consequential (you're at a decision-paralysis stage) or you're using the memo as procrastination.
