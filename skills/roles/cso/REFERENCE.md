# CSO — REFERENCE

The strategy frameworks the CSO role draws on. Cited in `references.md`.

**Operator-stage focus.** The frames below assume a pre-$50M ARR company. Larger companies have additional concerns (multi-product portfolio strategy, capital allocation across BUs, global market entry) that this skill doesn't cover.

---

## 1. The strategy 3-question filter (Porter / Lafley & Martin)

Every strategic decision answers three questions:

```
1. WHERE TO PLAY
   - Which customers / segments
   - Which geographies
   - Which product categories
   - Which channels
   - Which problems we'll solve

2. HOW TO WIN
   - What's our unfair advantage in this space?
   - Why will we beat competitors here?
   - What's the wedge (the specific thing only we do)?

3. WHAT TO GIVE UP
   - Which customers we WON'T serve
   - Which features we WON'T build
   - Which partnerships we WON'T pursue
   - Which markets we WON'T enter
```

**If you can't answer all three, you don't have a strategy yet — you have aspiration.**

Operator default: write your strategy as one-paragraph answers to each of the three questions. Update annually.

## 2. The wedge (the operator's competitive question)

A wedge is the **specific thing only you do better than anyone else right now**. Different from value proposition (what customers buy) or differentiation (how you're different).

| Type of wedge | Example pattern |
|---|---|
| **Customer-segment wedge** | "We're the only [thing] purpose-built for [specific persona]" |
| **Workflow wedge** | "We're the only [thing] that fits into [specific existing workflow]" |
| **Integration wedge** | "We're the only [thing] that already works with [specific tool customers use]" |
| **Price wedge** | "We're the only [thing] under $X for [specific outcome]" — rare and risky |
| **Compliance wedge** | "We're the only [thing] that meets [specific regulatory requirement]" |
| **Geographic wedge** | "We're the only [thing] localized for [specific market]" |
| **Speed wedge** | "We're the only [thing] that [task] in <[time]]" |
| **Data wedge** | "We have proprietary data / network that no one else has" |

**Wedge tests:**
- Is it true? (Not aspirational — verifiable.)
- Is it specific? ("AI-powered" isn't a wedge; "AI council with 9 role personas" might be.)
- Is it valuable to the customer?
- Is it defensible for 2-3 years?

**Most operators don't have a wedge. They have a "we do X but better" claim.** That's not a wedge; it's a tagline. Real wedges are narrow + true + defensible.

## 3. Competitive analysis — the operator's frame

**Don't waste cycles on competitive analysis.** The instinct to deeply study competitors is procrastination 80% of the time. The right amount of competitor analysis at operator-stage:

```
Per competitor:
- 1-2 hours of research initially
- Quarterly check-in (have they shipped something material?)
- Read their blog / changelog / pricing page when it changes

What to look at:
- Their pricing and packaging (vs yours)
- Their stated positioning (vs yours)
- Customer reviews / case studies (vs your customer feedback)
- Their team / fundraising (signals of trajectory)

What to skip:
- Reverse-engineering their codebase
- Estimating their revenue with elaborate modeling
- Deep employee LinkedIn diving
- Detailed feature-by-feature comparison matrices
```

Once you understand a competitor's position, you don't need to study them again until something material changes.

## 4. Where-to-play decisions

The most consequential strategic question for operator-stage companies: which slice of the market do we go after?

**Pick on these axes:**

| Axis | Question |
|---|---|
| Segment | Which kind of customer? (Size, industry, role, geography) |
| Use case | What specific job-to-be-done? |
| Distribution | How do they buy? (Self-serve, sales-led, channel, marketplace) |
| Position | Premium / mid-market / commodity? |
| Sequence | Which segment first? Which next? |

**Operator pattern:** start narrow, expand later. *"We're the [exact thing] for [exact persona] who [exact circumstance]"* beats *"We do productivity for everyone."*

**The land-and-expand sequencing:**

```
Land: Start with one tightly-defined ICP. Win them clearly.
Expand: Adjacent segments once the core is locked in.
  - Adjacent persona (same industry, different role)
  - Adjacent industry (similar workflow, different vertical)
  - Adjacent geography (same product, different market)
  - Adjacent use case (same persona, broader scope)
```

**Anti-pattern:** trying to land in 3 segments simultaneously. You'll dilute marketing, sales, and product across all three.

## 5. How-to-win — the unfair advantage frame

For each segment you've decided to play in, name your unfair advantage:

```
Unfair advantages can come from:

PRODUCT
- Superior feature(s) the segment values most
- Superior workflow integration
- Superior data / training set

DISTRIBUTION
- Existing relationship / channel access
- Proprietary acquisition method
- Lower CAC than competitors

OPERATIONS
- Faster shipping / iteration cycle
- Lower cost structure
- Higher customer success / retention

BRAND
- Founder/team credibility in the space
- Existing audience / community
- Trust in the segment

NETWORK
- Marketplace effects (more users = more value)
- Data network effects
- Direct network effects
```

**Reality check**: most operator-stage companies don't have a real unfair advantage yet. They have a *hypothesis* of unfair advantage. The work is to build the advantage by being unusually focused.

## 6. The "should we react to this competitor move" decision tree

Competitor ships a feature you don't have. Competitor announces a partnership. Competitor cuts prices. Decision tree:

```
1. Is this material to our customers?
   - If your customers haven't asked for it / aren't churning over it → ignore
   - If they ARE asking for it → continue

2. Does this change our wedge / positioning?
   - If our wedge is still defensible (we have something they don't) → adjust on the margin, don't pivot
   - If our wedge has been eaten → real strategic decision required

3. Can we credibly match / exceed in 3 months?
   - If yes: build it, but don't compromise other priorities
   - If no: accept the position, double down on our wedge

4. Is this signal of a category shift?
   - One competitor doing a thing = noise
   - Three competitors doing the same thing = signal worth investigating
```

**Most competitor moves don't warrant a response.** The companies that react to every competitor move don't have a strategy — they have a reaction function.

## 7. Partnership evaluation

A partnership has cost. Don't pursue unless the value is clear.

**The 4-question filter:**

```
1. STRATEGIC FIT — does this advance our where-to-play / how-to-win?
   - If yes, continue
   - If no (or "kinda"), decline

2. DISTRIBUTION VALUE — does this give us access to a real customer audience?
   - Specific number: how many qualified prospects do we reach via this partner?
   - If <100 qualified prospects, skip

3. EFFORT vs REWARD — what does this cost in engineering / GTM time?
   - 2 weeks: low bar
   - 1 quarter: needs strong yes on (1) and (2)
   - 6 months: only if this is strategically central

4. REVERSIBILITY — can we exit in 12-24 months without major cost?
   - If yes, low risk
   - If no (exclusivity, long-term commitment, partner's leverage), high bar
```

**Operator-stage default:** most partnerships fail to deliver. Default to no. Demand specifics on (1)-(4) before considering yes.

## 8. M&A framing (operator-stage, both directions)

### Acquiring (rare at operator-stage)

Operator-stage companies rarely acquire. When the question comes up:

```
- Acqui-hire: yes, if the team fills a critical gap and culture-fit is real
- Product acquisition: only if the product accelerates your wedge meaningfully
  AND integration cost is bounded
- Customer acquisition (buying customers via the acquired company): rarely 
  worth it; CAC math usually doesn't favor it
```

**Operator-stage M&A guidance:** if you're thinking about acquiring, you probably shouldn't. Build first.

### Being acquired

Operator-stage companies are sometimes approached. When the inquiry comes:

```
□ Don't sell prematurely (most founders regret selling too early)
□ Don't reject without numbers (knowing your acquisition value is useful)
□ Don't engage without an M&A advisor or banker (you'll get a better deal)
□ Don't talk publicly about the process (kills negotiating leverage)
□ Don't accept a deal that doesn't change your team's outcomes for the better
```

**Decision framework:**

| Trigger | Lean |
|---|---|
| Strategic acquirer offers >3× current valuation, lets team stay together | Likely yes |
| Random acqui-hire offer that doesn't significantly outpace your trajectory | Likely no |
| You're tired and want out | Be honest about that; the deal you sign in fatigue is rarely the best deal |
| Acquirer is hostile / aggressive in approach | Run, don't engage |
| Investor pressure to sell | Read the investor rights agreement carefully; don't be pressured |

## 9. Big-bet evaluation

For decisions involving multi-quarter commitment, run the big-bet framework:

```
THE BET STATEMENT
"We are betting that [condition], and if true, [outcome]. We're putting [resources] 
behind this for [duration]. If [leading indicator] doesn't show by [date], we 
will [action]."

THE INVERSION
What would have to be true for this to fail? List 3-5 failure modes.

THE EXPECTED VALUE
If successful, magnitude of upside?
If failed, magnitude of downside?
Probability of each?

THE COMMITMENT CURVE
At what milestones do we double-down? At what milestones do we walk?
```

**Operator-grade big-bet examples:**

- "We're betting that enterprise will adopt vertical AI agents before generalist agents..."
- "We're betting that the EU will require local AI sovereignty by 2027..."
- "We're betting that the legal industry will pay $50K/yr for AI workflow tools..."

Each is a 12-24 month bet with measurable leading indicators.

## 10. Market entry / market exit

### Entry

Before entering a new market (vertical, geography, segment):

```
□ Define the wedge for this specific market (different from your current wedge)
□ Identify the 5-10 first customers / accounts you'll target
□ Allocate resources (typically dedicated salesperson + dedicated marketer + product accommodations)
□ Set the success threshold (e.g., 3 paying customers in 6 months)
□ Set the kill threshold (e.g., <1 paying customer in 9 months → exit)
□ Determine the cost ceiling (most failed market entries spend 3-5× their budget)
```

### Exit (the harder discipline)

Most strategic mistakes are continuing to invest in markets that won't work. The exit discipline:

```
Triggers for exit consideration:
- Below kill threshold at the planned check-in
- Customer feedback shows we're solving the wrong problem
- Competitor consolidation in the market we're entering
- Resource pull-back from core business
- Strategic team consensus that this isn't working

The exit conversation:
- Don't blame individuals — strategic exits are strategic, not personal
- Allocate the freed resources deliberately (don't let them dissipate)
- Communicate honestly to affected customers / partners
- Document the lessons learned
```

## 11. Scenario planning (operator-grade)

For uncertain decisions, plan three scenarios (per Pierre Wack / Shell tradition):

```
SCENARIO A — Expected (50% probability)
[Your default operating plan]

SCENARIO B — Upside (25%)
[What we do if things break well — market grows 2x, big customer signs, etc.]

SCENARIO C — Downside (25%)
[What we do if things break badly — funding tightens, key person leaves, market shrinks]

For each:
- Triggers that signal which scenario we're in (by month 3 / 6)
- Actions to take if we recognize we're in it
- Resource implications

Cross-reference to runway model (CFO `templates/runway-model.md`) for downside cash impact.
```

**The downside scenario is the most important.** Most operators have a base plan and a vague optimism; few have a written "what we do when this goes wrong."

## 12. The strategy operating cadence

```
Annual: full strategy review
  - Update where-to-play / how-to-win
  - Define big bets for the year
  - Set objectives + key results

Quarterly: strategy check-in
  - Are we executing the strategy?
  - Has the market shifted (do we need to adjust)?
  - Are the big bets tracking?

Monthly: tactical adjustments
  - Course corrections within the strategy
  - Resource reallocation across initiatives

Weekly (CEO/founder): one strategic question
  - "What's the one decision that matters most this week?"
```

Strategy that doesn't live in an operating cadence is just a deck.
