# Pricing canvas — template

Use before any pricing change, new-tier launch, or repricing decision. Triangulates value, competitive, and cost anchors.

---

## Section 1 — The customer's outcome

```
For [target customer segment],
the outcome we deliver is [the specific job-to-be-done].
The dollar value of that outcome to the customer is approximately $[low] – $[high] per [time period].
Evidence: [interviews, willingness-to-pay studies, observed price points for adjacent solutions]
```

## Section 2 — Value-based price range

```
Lower bound: 10% of customer outcome value (commodity pricing)
Upper bound: 30% of customer outcome value (premium pricing)

→ Value-based range: $[      ] – $[      ] per [period]
```

If you can't fill in the dollar value of the customer outcome, **you are not ready to price.** Do 5 customer interviews first.

## Section 3 — Competitive anchor

| Competitor | Their price | What they include | What they're missing | What this implies |
|---|---|---|---|---|
| [   ] | $[ ] | [   ] | [   ] | [   ] |
| [   ] | $[ ] | [   ] | [   ] | [   ] |
| [   ] | $[ ] | [   ] | [   ] | [   ] |

**Rule:** don't price *at* competition. Price above with a reason, or below with a reason. "Same as competitor" is brand-erosive.

## Section 4 — Cost floor

```
Variable cost per customer:    $[      ] / [period]
Target gross margin:           [    ]%   # SaaS: ≥70%; physical: ≥40%
→ Cost-floor price:            $[      ]
```

Below the cost-floor, you're paying for the privilege of acquiring customers.

## Section 5 — Triangulate

```
Value-based range:    $[      ] – $[      ]
Competitive context:  $[      ] (with rationale)
Cost floor:           $[      ]

Recommended price:    $[      ]
Reasoning (one sentence): [why this price]
```

## Section 6 — Packaging / tiering

If you have multiple tiers, the canvas applies per tier. Common errors:

| Error | Fix |
|---|---|
| Linear pricing (Tier 2 = 2× Tier 1) | Use 1× / 3× / 10× — buyers self-select more cleanly |
| Same features, more usage | Add at least one capability gate per tier |
| No annual / multi-year discount | 15-20% off annual is industry standard; collect cash sooner |
| Free tier with no path to paid | Either remove free or instrument an explicit conversion trigger |

## Section 7 — Rollout plan

| Item | Decision |
|---|---|
| New price effective for new customers from | [date] |
| Existing customers — grandfathered or migrated? | [grandfathered / migrated at renewal / immediate migration] |
| Communication plan | [internal first, then sales, then customers via X channel on Y date] |
| Leading indicator (week 1) | [conversion rate on new pricing page] |
| Kill criterion | [if conversion drops >50% vs prior baseline, revert to prior pricing] |
| Decision date | [+30 days from rollout] |

## Section 8 — One-way vs two-way

```
Is this a one-way door?  [yes / no]
  - Grandfathering existing customers     → one-way
  - New price for new customers only      → two-way (default to this)
  - Removing a tier                       → one-way for existing tier members
  - Migrating existing customers up       → one-way
```

For two-way pricing changes, the canvas is a planning aid. For one-way changes, treat it as an engage-mode artifact and walk through it with a co-decider.
