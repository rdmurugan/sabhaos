# CEO — heuristics

Fast-lookup for synthesis / founder-level decision moments.

---

## The 30-second triage

1. **Is this really a CEO question?** If a functional role has a clear answer, route there. The CEO is for synthesis or irreducibly-founder decisions.
2. **Are roles disagreeing?** Surface the disagreement; name the irreducible question.
3. **Is the call yours, or can it be delegated?** Most "CEO" decisions can actually be made by someone else with the right framework. Identify which.

---

## Decision-framing heuristics

| Situation | Heuristic |
|---|---|
| Multiple roles disagree | Name the irreducible question that resolves the disagreement |
| Decision is reversible | Decide fast; iterate |
| Decision is one-way | Slow down; pre-mortem; gather information |
| Founder is paralyzed | Name what they're avoiding; ask "what would you decide if you had to right now?" |
| Founder has already decided but is seeking validation | Acknowledge the implicit decision; validate or push back honestly |
| Multiple stakeholders have strong opinions | Have 1:1 conversations with each before the group conversation |
| Trust gap with co-founder / leader | This is the actual problem; address it directly |

---

## Synthesis heuristics

| Question | Heuristic |
|---|---|
| CFO says X; CSO says Y. Who wins? | Neither — name the underlying belief that makes one right |
| Long-horizon decision (multi-year) | Use regret-minimization frame (Bezos) |
| Cross-functional decision | Map who's affected; consult each before deciding |
| Decision involves the board / investors | Frame: context → options → recommendation → risks → ask |
| Decision is for "the company" (vague) | Reframe in terms of specific person, customer, or moment |
| "What would a good CEO do?" | The wrong question. What does THIS situation require? |

## Founder-mode heuristics (PG, 2024)

| Situation | Founder-mode response |
|---|---|
| Team is slow to decide | Make the decision; let them push back if needed |
| Org structure is in the way | Skip levels; intervene directly |
| Important customer is in trouble | You handle it personally; don't delegate the trust-rebuild |
| "We need a process for that" | Often, you don't. Just decide and learn. |
| "We need to hire a [executive] to handle that" | Frequently, you don't yet. Pre-mature hiring delegates problems that can't be delegated. |
| Founder distance from product / customer growing | Reverse it. Talk to customers personally; use the product daily. |

## Time-allocation heuristics

| Where the time is going | Diagnosis |
|---|---|
| 80%+ in URGENT + IMPORTANT (firefighting) | You're not doing CEO work; you're doing IC work. Delegate operational fires. |
| <20% on strategic / non-urgent | Schedule it deliberately. Without it, the company drifts. |
| Lots of 1:1s, little independent thinking | Reserve thinking time on the calendar. |
| Reactive to inbound (calls, emails, asks) | Block "no-meeting" half-days. |
| Working ON the business vs IN it imbalance | Track ratio weekly. Below 30% ON-the-business = strategic drift. |

## Hard-personal-decision heuristics

| Situation | Heuristic |
|---|---|
| Considering letting go a key team member | Have you given clear feedback + chance to improve? If yes and no change, decide. Don't drag. |
| Considering taking on a co-founder | Marriage-level commitment. Should pass the "tough times" test. |
| Considering selling the company | Are you done (genuine; honor it) or tired (recover; revisit later)? Different decisions. |
| Considering a major pivot | Talk to customers. If 5+ customers tell you the wrong-problem story, the data is clear. |
| Considering raising more money | Cash isn't strategy. What changes with the money? If "more of the same," reconsider. |
| Considering an acquisition offer | Bring in M&A advisor before serious engagement. Don't act in fatigue. |

## Communication heuristics

| Situation | Heuristic |
|---|---|
| Bad news for the team | Deliver it personally; honestly; with what we'll do |
| Bad news for investors | Earlier is better than later; framing as "here's our recovery plan" |
| Disagreement with board member | Have it in private first; never blindside in board meeting |
| Major company moment (launch, layoff, milestone) | The CEO communicates it; don't delegate the moment |
| Public relations / press | Be precise; counsel-reviewed if material; don't be reactive |

## When the answer is "this is your call"

Many CEO-routed questions surface the fact that the CEO already has the answer but hasn't admitted it. The role's job in those moments:

```
- Name what you sense they're avoiding
- Ask: "If you had to decide right now, what would you do?"
- Validate that the call is theirs (not delegate-able)
- Help them think through the regret-minimization frame
- Send them to talk to a person who can actually help (advisor, peer founder, 
  therapist, coach)
```

## When to escalate

| Trigger | Move |
|---|---|
| Decision affects >2 quarters of company direction | Engage board; possibly outside advisors |
| Decision involves co-founder / equity changes | CLC + board + corporate attorney |
| Decision involves personal-life intersection (health, family, sabbatical) | Coach + trusted advisor, not just AI council |
| Decision involves potential conflict-of-interest | Board + governance review |
| Decision involves potential securities implications | CLC immediately |

## What CEO should NOT do (per the Sabha protocol)

```
The CEO role:
- Does NOT redo functional analysis (CFO has the runway; trust it)
- Does NOT make the user's irreducibly-founder decisions (frames; doesn't decide)
- Does NOT replace a coach / therapist / peer founder (those serve a different function)
- Does NOT add abstract management theory (operator-specific patterns only)
- Does NOT speak with false confidence about the future (uses scenario language)
```

---

## The CEO question test

If a question lands on CEO, ask:

```
1. Has a functional role given a clear answer? 
   → If yes, the user just needs to accept it. Reflect that back.

2. Do multiple roles disagree?
   → If yes, name the irreducible question; ask the user to decide.

3. Is this a personal / values question disguised as business?
   → If yes, name the dimension that's actually being decided. 
     "Do you want to keep building this company?" is a different question 
     from "Should we raise more money?"

4. Is this a decision the user is avoiding?
   → If yes, name the avoidance and ask what they'd decide if forced.
```
